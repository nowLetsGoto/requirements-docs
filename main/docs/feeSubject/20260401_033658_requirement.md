一、 功能导航页面路径：费用管理 —— 费用科目
核心结构：左侧树形结构 + 右侧列表

二、 字段级详细规则表
字段名称	控件类型	必填	校验规则 & 长度限制 & 重复性说明	默认值/枚举值/自动生成规则
费用科目编码	文本框	是	最大20字符，不可重复提示语：费用科目编码已存在	默认值：KM+年+月+日+4位数字自动生成
费用科目名称	文本框	是	最大50字符，不可重复提示语：费用科目名称已存在	无默认值
费用类型	下拉框	是	枚举值：管理费、销售费、财务费、其他费	默认值：管理费
备注	文本框	否	最大200字符	无默认值

三、 核心业务逻辑 (逻辑深度解构)
1. 层级与结构逻辑
层级上限：最多5级
树形联动：点击左侧树形结构节点，右侧列表展示该节点下所有子费用科目

2. 共享与级联约束
父子级级联：父级共享属性为“共享”，则子级必须共享，强制约束
同级一致性：同级科目在共享属性上必须保持一致性
同步修改逻辑：修改父级共享属性后，下级自动同步更新共享属性

四、 操作行为与权限约束
新增
触发位置/条件：点击新增按钮，弹窗初始化带出费用类型默认值为管理费
校验提示语：费用科目编码已存在，费用科目名称已存在
编辑
单选，费用科目编码不可编辑
置灰字段：费用科目编码
报错提示语：费用科目编码不可修改
更新记录：保存后更新费用科目名称、费用类型、备注
删除
不可删除条件：被其他业务模块关联时不可删除
提示文案：该费用科目已被使用，无法删除
删除方式：逻辑删除
停用/启用
停用前置条件：无子级启用状态
状态变更逻辑：停用后子级自动停用，启用时需手动启用子级

五、 交互细节补充
弹窗交互：取消时保留已输入信息，确认保存时报错样式为红色边框+提示文字
预置数据：系统默认存在费用科目名称列表，包含：管理费、销售费、财务费、其他费

变量映射：费用管理 / 费用科目，存储路径：main/docs/feeSubject
